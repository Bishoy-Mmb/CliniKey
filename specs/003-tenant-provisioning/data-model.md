## Entity: Clinic

**Storage**: `shared.clinics`  
**Aggregate root**: Yes  
**Existing type**: `CliniKey.Domain.Entities.Clinic`

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `Id` | `Guid` | PK, generated by domain | Tenant ID and JWT `tenant_id` value |
| `Name` | `string` | Required, max 200 | Not unique |
| `Phone` | `PhoneNumber` | Required, unique | Egyptian mobile format via existing value object |
| `Address` | `string` | Required, max 500 | Clinic contact address |
| `SchemaName` | `string` | Required, unique, max 63, immutable | PostgreSQL schema name, e.g. `tenant_ab12cd34` |
| `Status` | `ClinicStatus` | Required | `Active`, `Inactive`, `Suspended` |
| `ProvisioningStatus` | `TenantProvisioningStatus` | Required | `Pending`, `Provisioning`, `Provisioned`, `Failed`, `Migrating` |
| `SchemaHealthStatus` | `TenantSchemaHealthStatus` | Required | `Unknown`, `Healthy`, `Missing`, `MigrationPending`, `Unhealthy` |
| `CurrentMigration` | `string?` | Nullable, max 150 | Last applied tenant migration |
| `LastSchemaVerifiedAtUtc` | `DateTime?` | Nullable | Updated by health checks |
| `DeactivatedAtUtc` | `DateTime?` | Nullable | Set on deactivate/suspend |
| `DeactivatedByUserId` | `Guid?` | Nullable | Platform operator user ID when available |
| `CreatedAtUtc` | `DateTime` | Required | From `AggregateRoot` |
| `UpdatedAtUtc` | `DateTime?` | Nullable | From `AggregateRoot` |

**Behavior**:

- `Create(name, phone, address, schemaName, clock)` returns `Result<Clinic>`.
- `MarkProvisioning()`, `MarkProvisioned(migration)`, `MarkProvisioningFailed(reason)` update provisioning state.
- `Activate()` moves `Inactive` or `Suspended` to `Active`.
- `Deactivate(operatorUserId)` moves `Active` to `Inactive`, preserving schema/data.
- `Suspend(operatorUserId)` blocks access for operational reasons while preserving data.
- `UpdateContact(phone, address)` validates and updates contact fields.
- `MarkSchemaHealth(status, currentMigration, verifiedAtUtc)` records health checks.

**Validation**:

- Name is required and <= 200 chars.
- Phone must satisfy `PhoneNumber.Create`.
- Phone is unique across `shared.clinics`.
- Address is required and <= 500 chars.
- Schema name is generated by the system, immutable, <= 63 chars, lowercase, and matches PostgreSQL identifier constraints.

**Domain events**:

- `ClinicProvisionedEvent`
- `ClinicActivatedEvent`
- `ClinicDeactivatedEvent`
- `ClinicContactUpdatedEvent`

## Entity: TenantSchemaHealth

**Storage**: `shared.tenant_schema_health`  
**Aggregate root**: No, infrastructure-facing registry record

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `ClinicId` | `Guid` | PK/FK to `shared.clinics` | One health row per clinic |
| `SchemaName` | `string` | Required, unique | Mirrors `Clinic.SchemaName` for quick checks |
| `Status` | `TenantSchemaHealthStatus` | Required | Current health state |
| `CurrentMigration` | `string?` | Nullable, max 150 | Latest applied tenant migration |
| `ExpectedMigration` | `string?` | Nullable, max 150 | Latest migration known by app |
| `LastVerifiedAtUtc` | `DateTime?` | Nullable | Health-check timestamp |
| `FailureReason` | `string?` | Nullable, max 1000 | Last migration/health error summary |

**Notes**: This table can be implemented as a separate table or folded into `shared.clinics`. A separate table is preferred if health checks are updated frequently to avoid noisy writes to the clinic aggregate row.

## Entity: TenantProvisioningAuditLog

**Storage**: `shared.tenant_provisioning_audit_logs`  
**Aggregate root**: No

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `Id` | `Guid` | PK | Generated per operation |
| `ClinicId` | `Guid?` | Nullable FK to `shared.clinics` | Null only when provisioning fails before clinic creation |
| `SchemaName` | `string?` | Nullable, max 63 | Target schema |
| `Operation` | `string` | Required, max 100 | `Onboard`, `CreateSchema`, `ApplyMigrations`, `Activate`, `Deactivate`, `HealthCheck` |
| `Status` | `string` | Required, max 50 | `Started`, `Succeeded`, `Failed` |
| `Message` | `string?` | Nullable, max 1000 | Human-readable summary |
| `OperatorUserId` | `Guid?` | Nullable | Platform operator when available |
| `OccurredAtUtc` | `DateTime` | Required | From `TimeProvider` |

**Notes**: Audit rows support FR-014 and operational support. They are append-only.

## Enum: ClinicStatus

| Value | Meaning |
|-------|---------|
| `Active` | Tenant users may access operational endpoints |
| `Inactive` | Tenant users are blocked; data is preserved |
| `Suspended` | Tenant users are blocked due to operational/security concern; data is preserved |

---

## Enum: TenantProvisioningStatus

| Value | Meaning |
|-------|---------|
| `Pending` | Clinic record exists but provisioning has not started |
| `Provisioning` | Schema creation/migration is in progress |
| `Provisioned` | Schema exists and migrations are current |
| `Failed` | Provisioning failed and requires operator attention |
| `Migrating` | Existing tenant schema is being upgraded |

---

## Enum: TenantSchemaHealthStatus

| Value | Meaning |
|-------|---------|
| `Unknown` | Health has not been checked yet |
| `Healthy` | Schema exists and is at expected migration |
| `Missing` | Registry points to a schema that does not exist |
| `MigrationPending` | Schema exists but is behind expected migration |
| `Unhealthy` | Schema exists but failed validation or migration |

## Tenant Operational Tables

**Storage**: `tenant_*` schemas only

Tenant migrations create and update:

- `patients`
- `appointments`
- `treatment_plans`
- `treatment_items`
- `invoices`
- `invoice_lines`
- `payments`
- `__EFMigrationsHistory`

Operational entities must not be created in `public` or `shared`.
