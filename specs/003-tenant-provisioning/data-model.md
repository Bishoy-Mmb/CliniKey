## Entity: Tenant

**Storage**: `shared.tenants`
**Aggregate root**: Yes
**Type**: `CliniKey.Domain.Entities.Tenant`

| Field | Type | Constraints | Notes |
| --- | --- | --- | --- |
| `Id` | `Guid` | PK, generated by application | Tenant/practice ID and JWT `tenant_id` value |
| `Name` | `string` | Required, max 200 | Practice display name |
| `SchemaName` | `string` | Required, unique, max 63, immutable | PostgreSQL schema name, e.g. `tenant_<guid>` |
| `Status` | `TenantStatus` | Required | `Active`, `Inactive`, `Suspended` |
| `ProvisioningStatus` | `TenantProvisioningStatus` | Required | `Pending`, `Provisioning`, `Provisioned`, `Failed`, `Migrating` |
| `SchemaHealthStatus` | `TenantSchemaHealthStatus` | Required | `Unknown`, `Healthy`, `Missing`, `MigrationPending`, `Unhealthy` |
| `CurrentMigration` | `string?` | Nullable, max 150 | Last applied tenant migration |
| `LastSchemaVerifiedAtUtc` | `DateTime?` | Nullable | Updated by provisioning/migration health checks |
| `DeactivatedAtUtc` | `DateTime?` | Nullable | Set on deactivate/suspend |
| `DeactivatedByUserId` | `Guid?` | Nullable | Platform operator user ID when available |
| `CreatedAtUtc` | `DateTime` | Required | From `AggregateRoot` |
| `UpdatedAtUtc` | `DateTime?` | Nullable | From `AggregateRoot` |

**Behavior**:

- `Create(id, name, schemaName, clock)` returns `Result<Tenant>` and raises `TenantCreatedEvent`.
- `MarkProvisioning()` records provisioning start.
- `MarkProvisioned(migration)` sets `Provisioned`, `Healthy`, and current migration.
- `MarkProvisioningFailed()` marks provisioning and schema health failed.
- `Activate()` and `Deactivate(operatorUserId)` control tenant-level access.
- `Suspend(operatorUserId)` blocks tenant access for operational/security reasons.
- `MarkSchemaHealth(status, currentMigration, verifiedAtUtc)` records migration/health checks.
- `AddClinic(clinic)` associates a branch with the tenant aggregate.

## Entity: Clinic

**Storage**: `shared.clinics`
**Aggregate root**: Yes
**Type**: `CliniKey.Domain.Entities.Clinic`

| Field | Type | Constraints | Notes |
| --- | --- | --- | --- |
| `Id` | `Guid` | PK, generated by application | Branch/location ID |
| `TenantId` | `Guid` | FK to `shared.tenants.id` | Owning practice |
| `Name` | `string` | Required, max 200 | Branch display name |
| `Phone` | `PhoneNumber` | Required, unique | Egyptian mobile format |
| `Address` | `string` | Required, max 500 | Branch contact address |
| `Status` | `ClinicStatus` | Required | Branch lifecycle status |
| `DeactivatedAtUtc` | `DateTime?` | Nullable | Set on branch deactivate/suspend |
| `DeactivatedByUserId` | `Guid?` | Nullable | Platform operator user ID when available |
| `CreatedAtUtc` | `DateTime` | Required | From `AggregateRoot` |
| `UpdatedAtUtc` | `DateTime?` | Nullable | From `AggregateRoot` |

Clinic does not own schema name, provisioning status, schema health, or current migration.
Those belong to `Tenant`.

## Enums

### TenantStatus

| Value | Meaning |
| --- | --- |
| `Active` | Tenant users may access operational endpoints if also provisioned and healthy |
| `Inactive` | Tenant users are blocked; data is preserved |
| `Suspended` | Tenant users are blocked due to operational/security concern; data is preserved |

### ClinicStatus

| Value | Meaning |
| --- | --- |
| `Active` | Branch/location is active |
| `Inactive` | Branch/location is inactive |
| `Suspended` | Branch/location is suspended |

### TenantProvisioningStatus

| Value | Meaning |
| --- | --- |
| `Pending` | Tenant row exists before provisioning starts |
| `Provisioning` | Schema creation/migration is in progress |
| `Provisioned` | Schema exists and baseline migration succeeded |
| `Failed` | Provisioning failed |
| `Migrating` | Future state for active tenant migration runs |

### TenantSchemaHealthStatus

| Value | Meaning |
| --- | --- |
| `Unknown` | Not yet verified |
| `Healthy` | Schema exists and current migration is acceptable |
| `Missing` | Schema does not exist |
| `MigrationPending` | Schema exists but is behind expected migration |
| `Unhealthy` | Schema cannot safely serve tenant traffic |

## Relationships

```text
public.AspNetUsers.TenantId -> shared.tenants.id

shared.tenants.id -> shared.clinics.tenant_id
shared.clinics.id -> shared.clinic_dentists.clinic_id
shared.dentists.id -> shared.clinic_dentists.dentist_id

shared.tenants.schema_name -> tenant_<guid> schema
```

Tenant operational tables do not store tenant IDs for isolation; they live in the
tenant schema selected by request resolution.
